Detailed Description

A generic and dynamic container that expands automatically as elements are added. The Vector supports amortized constant time insertion and removal of elements , as well as constant time access.

C doesn’t allow data types to be passed as parameters. That means a generic container needs to manually manage memory in terms of the client element size, not client data type.

size_t vector_t::ele_size

Vector's data type. It has to be the returned value of 'sizeof(Vector's data type)'

The constructor requires that the client element size be identified up front.

Since it doesn’t (and will never) know the true data type, it needs the size so it at least knows how many bytes to replicate behind the scenes with every call to manage data.
